A design system reverse-engineered from one ERP dashboard.

This reference catalogs every reusable piece of the source interface — tokens, primitives, composed components, and the interaction patterns that connect them — so new screens can be built that feel native to the original product.

Flat, content-led surfaces

White cards on a soft grey canvas, separated by hairline borders rather than heavy shadow — elevation is implied, not dramatized.

One accent does the talking

Raspberry-pink marks the primary data series and calls to action; indigo, blue and green are reserved for secondary series and status.

Numbers are the heroes

Metrics are set large and bold with tabular figures; supporting labels stay small, grey, and quiet so the number reads first.

Behavior / 01

Patterns

Solutions for how groups of components behave together — not single elements, but the interactions that connect them.

Pattern 01

Series toggling

The Income / Expense / Net Profit checkboxes act as a visibility legend, not a filter form. Checking a box adds that series' bars and re-colors them in its accent; unchecking fades it back to the neutral surface tone. At least one series stays checked at all times.

Pattern 02

Range highlight + tooltip

Hovering or selecting a span of months brightens that range to full-opacity pink while the rest of the chart recedes to a neutral fill. A floating tooltip anchors above the midpoint of the selection, showing the aggregate value and date range.

Pattern 03

Stat card → drill-down

Every metric card ends in a quiet "Show More" text-link rather than wrapping the whole card in a click target — this keeps the number selectable/copyable and avoids accidental navigation while scanning.

Pattern 04

Carousel overview

Horizontally-paginated card rows (e.g. "Explore Project Overview") use a pair of round prev/next icon buttons docked top-right of the section header, rather than arrows on the cards themselves, keeping pagination separate from content.

Pattern 05

Responsive sidebar collapse

The top-left rail icon collapses the sidebar to icon-only at narrow widths; labels fade out while active-state background and icon stay, preserving wayfinding without text.

Pattern 06

Notification & presence dots

A single status dot vocabulary covers two meanings by position and color: top-right of the bell (pink) means unread, bottom-right of an avatar (green) means online — never combined on the same element.

Behavior / 02

Documentation

Usage guidance for applying this system consistently to new screens.

Component usage

ComponentUse forAvoid
Primary button (black)One decisive action per view, e.g. "Explore Account"More than one per screen — dilutes the action hierarchy
Pink accentThe single primary data series, checked states, and the one "hero" CTADecorative use, or applying it to more than one series at once
Stat cardA single countable metric with a year-over-year trendMulti-metric comparisons — use the chart card instead
List cardGrouping 2–6 navigational links under one record typeLong lists past ~6 items — paginate or link to a full index
TooltipContextual value on hover/selection inside chartsPersistent help text — use inline captions instead

Do & don't

Do
  • Pair every icon tile with a plain-text label, never icon alone
  • Keep one pink call-to-action visible per screen
  • Set numeric values in tabular figures so columns align
  • Use 1px borders before adding shadow
Don't
  • Mix more than one rounded-corner radius within a single card
  • Use red/pink for anything that isn't the primary series or a negative trend
  • Stack two outline buttons of equal visual weight as the main actions
  • Drop the divider between a card's metric and its footer actions

Content & voice

Labels name what the person controls, in plain sentence case — "Show More," "Filters," "New Task" — never system jargon. Section eyebrows ("One Year Statement") set context before the headline so the number that follows is never ambiguous. Trend badges always pair the percentage with a directional icon so meaning doesn't depend on color alone.

Accessibility

ContrastBody text (#6E6E78) on white passes AA at 14px; tertiary gray (#9C9CA6) is reserved for non-essential captions only
Hit targetsAll icon buttons are 40×40px minimum, even where the visible glyph is 15–18px
Focus stateInputs and buttons receive a 3px indigo ring at 12% opacity on keyboard focus, matching the active checkbox/select accent
Color independenceTrend badges and series legends always carry an icon or label alongside color
